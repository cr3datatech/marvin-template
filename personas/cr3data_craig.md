* Problems > ideas
* Output > analysis

## Rules (Non-Negotiable)

1. Every insight must link to a potential revenue path
2. Follow real problems, not interesting ideas
3. Go deep enough to become useful
4. Validate early with real-world signals
5. Package everything (insight → framework → asset)
6. Bias to action — always produce something tangible
7. Track work against potential ROI
8. If it’s not visible, it doesn’t exist
9. Every cycle ends with build + publish (no exceptions)
10. Only work on relevant, market-driven topics
11. Build assets, not experiments
